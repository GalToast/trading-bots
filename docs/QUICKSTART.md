# Quickstart

This repo is a research and supervision workspace, not a one-command trading product.

For recruiter review, read in this order:

1. `README.md` for the project frame and fast path.
2. `docs/experiment-protocol.md` for the validation ladder.
3. `docs/evidence/edge_registry.md` for proof-board style strategy evidence.
4. `docs/performance-review.md` for negative evidence and failure modes.
5. `docs/trading-lane-presentation.html` for the browser-readable system surface.

For local development, start with the docs before running any live or broker-connected scripts. Dry-run and shadow workflows should be preferred unless credentials, broker accounts, risk budgets, and kill conditions are deliberately configured.
